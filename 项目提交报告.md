## 二、功能说明

### 核心功能清单

1. **多角色 Agent 协作**：信息采集（Collector）、竞品分析（Analyst）、报告撰写（Writer）、质量检查（QC）四大 Agent 独立运行，通过 LangGraph StateGraph 协同工作
2. **结构化知识抽取**：定义竞品知识 Schema（功能矩阵、定价模型、用户情感、SWOT 分析、关键洞察），Pydantic 运行时校验确保输出一致性
3. **DAG 条件路由 + 真实反馈闭环**：Agent 间通过结构化 State 传递消息，QC 按问题类型精准打回（collector/analyst/writer），支持最多 3 轮迭代，超限后降级输出（非伪闭环）
4. **全链路信息溯源**：每条 DataPoint 携带 source\_url 和 confidence 字段，报告末尾自动汇总所有数据来源，支持一键跳转原始链接
5. **可观测性**：每次 LLM 调用全量记录（agent\_name、step、prompt、token 消耗、耗时），前端提供 Agent 决策回放时间线
6. **双模式运行**：在线模式（实时搜索 + 网页抓取）与离线模式（预采集数据演示），API 不可用时仍可完整演示

### 端到端使用流程

1. 用户在前端界面（Streamlit）或 CLI 输入竞品分析需求：指定目标产品（如 TRAE）、竞品列表（如 Cursor、GitHub Copilot）、分析维度（功能对比、定价模型、用户评价、SWOT 分析）
2. 系统通过 LangGraph 自动启动 DAG 任务流转，首先调用 Collector Agent，从 DuckDuckGo 公开渠道多源搜索竞品信息，并对前 3 条搜索结果抓取页面正文
3. 采集数据经过 LLM 结构化提取，符合预定义 Pydantic Schema（DataPoint 含 field/value/source\_url/source\_type/confidence）后写入 WorkflowState，传递至 Analyst Agent
4. Analyst Agent 消费采集数据，执行功能矩阵评分（0-3 分，含 heuristic 降级路径）、SWOT 分析、用户情感分析（-1\~1 评分）、定价提取、关键洞察生成，产出结构化 AnalysisResult
5. Writer Agent 将分析结果按 8 章节模板（执行摘要/竞品概览/功能矩阵/定价对比/用户评价/SWOT/结论建议/数据来源）组装为完整 Markdown 报告，自动注入溯源引用
6. QC Agent 对全链路产物执行五层校验（章节完整性/来源校验/占位符检查/内容长度/LLM 语义校验），产出 QCVerdict
7. 若 QC 通过（pass\_=True），输出最终报告；若未通过，qc\_router 按 issue.target\_agent 精准打回对应 Agent 重新执行，最多 3 轮；超出上限则降级输出当前最优结果
8. 用户在前端查看完整报告，支持溯源链接点击跳转、人工修正文本后触发重新质检、Agent 决策过程全程回放

### 大模型 / AI 能力使用说明

**模型调用：**

- 主模型：Doubao-Seed-2.0-lite（doubao-seed-2.0-lite-251015）
- 调用方式：直接API调用，课题成员共享
- 用途：
  - Collector：搜索查询自适应生成 + 网页内容结构化提取（JSON 格式输出）
  - Analyst：功能矩阵评分（0-3）/ SWOT 分析 / 用户情感分析（-1\~1）/ 定价提取 / 关键洞察
  - Writer：执行摘要与结论建议的 LLM 增强生成
  - QC：LLM 语义一致性校验（数据矛盾检测、逻辑一致性、引用遗漏检查）

**Agent 设计：**

- 编排框架：LangGraph StateGraph（DAG 式任务编排，显式定义节点与边，支持多条 conditional edges）
- 通信协议：Pydantic Schema + TypedDict State 传递（非纯自然语言，Agent 间通过结构化 dict 传递数据）
- 反馈机制：QC Agent 五层校验（规则 + LLM + 溯源）→ QCVerdict.{pass\_, issues, target\_agent} → qc\_router 精准路由打回 → 最多 3 轮迭代 → 超限降级输出
- Prompt 工程：每个 Agent 独立 Prompt 模板（`config/prompts/`），含强制约束（如 Collector 必须 "ONLY use information from provided search results. NEVER invent data."）

**关键机制：**

- 幻觉抑制：三层防线——(1) Collector Prompt 强制仅从搜索结果提取；(2) QC LLM 校验抽样 20% 数据点验证事实性；(3) source\_url 为强制字段
- JSON 安全解析：`safe_json_parse()` 多层清理（去除 markdown 代码块 → JSON 解析 → 正则提取 → 失败则 LLM 自修复）
- 容错降级：搜索失败 → 离线预采集数据回退；LLM 评分失败 → heuristic 评分；JSON 解析失败 → 空结构

### 关键工程难点与解决方案

| 难点                    | 解决方案                                                                                                                                                                                                                                                       | 代码位置                                                                    |
| --------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------- |
| **Agent 间结构化通信与状态管理** | 基于 LangGraph StateGraph 的 TypedDict 共享 State，定义全局 WorkflowState（含 collected\_data/analysis\_result/report/qc\_result 等字段）。每个 Agent 产出 Pydantic Model → model\_dump() 写入 State → 下游 Agent 读取后 Pydantic(\*\*data) 重建。避免了自然语言解析的不确定性。                         | `src/orchestration/state.py` `src/orchestration/graph.py`               |
| **真实反馈闭环的无限循环风险**     | 设计三层防护：(1) 通过 `qc_router` 按 issue.target\_agent 和 severity 精准路由，而非全量重跑；(2) Collector 被打回时通过 collection\_hints 仅重采缺失维度，节省 API 调用；(3) 设置硬上限 MAX\_QC\_ITERATIONS=3，超限后状态标记为 completed\_degraded，输出当前最优结果并标注置信度。经全链路测试验证，得分逐轮上升（0.40→0.65→降级），闭环真实可触发。         | `src/orchestration/router.py` `config/settings.py`                      |
| **长文本信息的结构化提取与幻觉抑制**  | 设计分层 Schema + 分块处理策略：(1) Collector 对每个竞品独立采集，页面正文截断至 8000 字符，避免超长上下文；(2) Prompt 强制 "ONLY use information from provided search results. NEVER invent data."，并输出 JSON 格式；(3) QC 通过 `_llm_check()` 抽样 20% 数据点与原始采集结果交叉验证；(4) 三层安全解析（直接 JSON → LLM 自修复 → 抛异常）。 | `src/agents/collector.py` `src/agents/qc.py` `src/schema/validators.py` |

### 项目亮点 / 创新点

**亮点 1：真实反馈闭环——非伪闭环，质量逐轮提升**

QC Agent 不仅是格式检查，而是五层深度校验（规则 + LLM + 溯源），打回后针对性重做（仅重采缺失维度，非全量重跑），验证得分逐轮上升（实测 0.40 → 0.65），3 轮超限后降级输出而非伪闭环。区别于常见的「一次性质检」或「循环但不收敛」的方案。

**亮点 2：全链路可追溯——每条结论可追问**

从 DataPoint 到 FinalReport，每个数据点强制携带 source\_url 和 confidence 字段；报告末尾自动汇总编号来源列表，支持一键跳转原始链接；前端提供 Agent 决策回放时间线，评委可完整查看「谁在什么时间、用了什么 Prompt、输出了什么」的全过程。

**亮点 3：Schema 驱动一致性——可跨行业复用**

预定义竞品知识 Schema（CollectedData → AnalysisResult → FinalReport）以 Pydantic 强类型实现运行时校验，确保 Agent 输出格式统一。支持自定义竞品和维度（`--competitors "飞书,钉钉" --dimensions "功能对比,定价"`），可快速迁移至其他行业场景，具备前瞻性扩展设计。
