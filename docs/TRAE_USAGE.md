## 2. 深度协作模式

### 2.1 分工原则

```
┌─────────────────────────────────────────────────────┐
│                   人工决策（Human）                    │
│  · 系统架构设计（四层架构）                             │
│  · 技术选型（LangGraph vs CrewAI）                     │
│  · 关键设计决策（Pipeline Pattern、反馈闭环）            │
│  · 答辩策略与文档框架                                  │
├─────────────────────────────────────────────────────┤
│                   AI 执行（TRAE AI）                   │
│  · 全部代码实现（~2,500 行 Python）                     │
│  · 单元测试编写（15 条用例）                            │
│  · Code Review 与 Bug 修复                            │
│  · 6 份技术文档撰写                                    │
│  · Streamlit UI 设计与实现                             │
└─────────────────────────────────────────────────────┘
```

## 4. AI 协作亮点

### 4.1 复杂逻辑实现
- **QC 反馈闭环**：100 行条件路由逻辑由 AI 一次性生成并自测通过
- **Web Scraper**：HTML→Markdown 转换 + 中文编码处理
- **安全 JSON 解析**：三层降级策略（直接解析→LLM 修复→抛异常）

### 4.2 自动化测试
- AI 独立编写 15 条测试，覆盖 Agent、编排、路由全链路
- 测试驱动开发：部分模块先写测试再写实现

### 4.3 文档自动化
- AI 根据代码自动生成评分对照矩阵（SCORING_MATRIX.md）
- 架构图从实际代码中提取节点和边自动绘制

### 4.4 UI 设计
- 深色/浅色主题切换（200+ 行 CSS）
- Agent 工作记录卡片（含 Token 消耗、耗时可视化）
